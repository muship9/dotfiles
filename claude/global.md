# Claude Global Configuration

## 基本設定

### 言語設定
- **デフォルト言語**: 日本語
- **技術用語**: 英語を維持（例: commit, branch, function）
- **エラーメッセージ**: 原文（英語）を提示後、日本語で説明

### 応答スタイル
- **簡潔性重視**: 冗長な説明を避ける
- **実装優先**: 説明より実際のコード・コマンドを重視
- **確認最小化**: 明確な指示の場合は確認せず実行

## プロジェクト横断の規約

### コミットメッセージ
- **形式**: `type(scope): summary`
- **type**: feat, fix, docs, style, refactor, test, chore
- **言語**: 英語（簡潔に）

### コード規約
- **インデント**: 各言語・プロジェクトの既存スタイルに従う
- **命名規則**: 
  - Python/Ruby: snake_case
  - JavaScript/TypeScript: camelCase
  - Go: camelCase/PascalCase（公開度による）
- **コメント**: 必要最小限、なぜ（Why）を説明

### ファイル操作
- **編集優先**: 新規作成より既存ファイルの編集を優先
- **ドキュメント**: 明示的に要求されない限り作成しない
- **バックアップ**: 破壊的変更前に確認

## 開発環境

### 標準ツール
- **エディタ**: Neovim (nvim)
- **シェル**: Zsh
- **ターミナル**: WezTerm
- **バージョン管理**: Git

### パッケージマネージャー
- **macOS**: Homebrew
- **Node.js**: npm/pnpm
- **Python**: pip/poetry
- **Ruby**: gem/bundler
- **Go**: go mod

## セキュリティ

### 機密情報の扱い
- **環境変数**: `.env` ファイルや `~/.secrets/` に格納
- **APIキー**: 絶対にコミットしない
- **パスワード**: 平文で保存しない

### Git操作
- **確認事項**:
  - 機密情報が含まれていないか
  - `.gitignore` が適切に設定されているか
  - プッシュ先が正しいか

## トラブルシューティング優先順位

1. **エラーメッセージを正確に読む**
2. **既存のドキュメント・README確認**
3. **関連ファイルの設定確認**
4. **依存関係の確認**
5. **クリーンインストール試行**

## 特殊な指示

### デバッグ時
- ログ出力は最小限に
- 問題の根本原因を特定してから修正
- 一時的な回避策には `# FIXME:` コメント追加

### パフォーマンス
- 大量ファイル操作時は `fd` や `rg` を優先
- バッチ処理可能なものはまとめる
- キャッシュを適切に活用

## プロジェクト固有の設定

プロジェクトルートに `CLAUDE.md` がある場合、そちらの設定を優先する。
競合する場合の優先順位：
1. プロジェクトの `CLAUDE.md`
2. このグローバル設定
3. Claudeのデフォルト動作
